## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in Megradea_Mobile_App, please report it to us privately. This gives us time to fix the issue before it can be exploited.

### **Where to Report**

**Email**: wandende.security@gmail.com ← replace with your real email

**Subject line**: `[SECURITY] Vulnerability in Megradea_Mobile_App`

For highly sensitive issues, you can encrypt your email using our PGP key: [Add link if you have one, or remove this line]

### **What to Include**

To help us fix it faster, please include:
1. **Version**: Which version of the app is affected? `v0.1.0`, `main` branch, etc.
2. **Description**: What is the vulnerability and what impact could it have?
3. **Steps to Reproduce**: Detailed steps so we can see the bug ourselves
4. **Proof of Concept**: Screenshots, screen recording, or sample code if possible
5. **Your Android version**: Example: Android 13, Samsung A14
6. **Discovery credit**: How would you like to be credited if we publish a fix?

### **What to Expect From Us**

1. **Acknowledgment**: We will confirm we received your report within **72 hours**
2. **Updates**: We will send you status updates at least **every 7 days** until resolved
3. **Fix Timeline**: We aim to patch critical vulnerabilities within **14 days**
4. **Disclosure**: We ask that you give us **90 days** to fix the issue before any public disclosure
5. **Credit**: If you want, we will publicly thank you in the release notes after the fix is published

### **Out of Scope**

The following are not considered security vulnerabilities:
- Bugs that require physical access to an unlocked device
- Social engineering attacks
- Issues in third-party libraries - report those upstream
- App crashing without security impact

### **Safe Harbor**

We support responsible disclosure. If you follow this policy, we will not pursue legal action against you for your research.

Thank you for helping keep Megradea_Mobile_App and our users safe.
